🔌 injectCrawlResult() API 說明
Claude 完成 Chrome MCP 爬蟲後，呼叫此函數注入結果。注入後資料自動儲存，其他人開啟工具即可看到。
⚠️ PTT / 平台無資料的正確寫法（v12 修正）
// ✅ 正確：找不到資料 → 明確標示 null，不給假評分
social: {
  ptt: null, // 無結果 → 不計分、顯示「⚠️ 無資料」
  dcard: { score: 72, notes: "有多篇評比討論", sourceUrl: "..." }
}

// ❌ 錯誤（v11 bug）：無資料卻給 ★★★★★
ptt: { score: 100, notes: "搜尋結果為零" } // 不可這樣寫

📖 評分維度說明
所有分數 1–10，最終加權得 0–100 分
70+ 推進 ／ 50–69 條件評估 ／ 49 以下不採用
維度（佔分）資料來源無資料時處理
市場規模（×20）Google Trends + 各平台銷量標示無資料，分數保守估 5
需求強度（×20）重複購買率、社群討論趨勢同上
競爭密度（×20）各平台競品件數（越少=越高分）同上
差異化空間（×20）認證/品牌/功能唯一性同上
作業複雜度（×10）物流/冷鏈/法規需求依商品特性填入
毛利可行性（+10）依填入成本自動計算未填成本 = 0 分
PTT 無結果—⚠️ 標示無資料，不計入評分（v12 修正）
